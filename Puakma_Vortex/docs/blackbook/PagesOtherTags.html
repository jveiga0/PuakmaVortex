Path

Renders as path to the application. This is a handy tag useful for making application portable tags. For example you might use this code to reference an image, eg <img src="<P@Path @P>/myimg.gif?OpenResource”>. The path tag supports a value attribute to return only a part of the URI. For example for the URI "/apps/crm.pma" the tag will return "apps". Possible values are: Group, Application, Action, DesignElementName, Parameters, FileExt

Example:
<P@Path @P> renders for example to "/puakma.pma"

User name

Inserts the current user’s name in the page.

Attributes
value Format of the name. Valid values are "cn", "common", "canonical". All other values will result in the abbreviated name being inserted.
Example:
<P@UserName value="cn" @P> renders to for example: "Bob Smith/ACME"

Version string

Inserts a html comment in the source denoting the Puakma server version.

Example:
<P@Version @P> displays something like <!-- Puakma v2.32 Build:303 - 30 October 2003 -->

View

This will render a table from the relational database.

Attributes
name Defines the name of the view
datasource Is the SQL query to populate data.
connectoin Is the database connection name as defined among the application's database connections. This parameter is optional in the case that there is only one database connection.
previoustext The text which appears on the previous button which allows user to jump to the previous view of table rows (according to order of rows).
nexttext The text which appears on the next button which allows user to jump to the next view of table rows (according to order of rows).
maxperpage Maximum rows populated in one view. This parameter is optional The default value means all rows.
Example:
<P@View Name="ViewName" datasource="SELECT * FROM SOMETABLE ORDER BY SomeColumn" xslstylesheet="browse.xsl" connection="conn_name" nexttext="Next" previoustext="Previous" maxperpage="25" ... @P>
