HTML P-tags

P-tags are used to place html fields on a page. Tornado Server automatically converts these p-tags into their html equivalents (xhtml compliant) which can be understood by web browsers. P-tags allow the programmer to represent a field with a single tag and that tag can be easily changed later to completely change the field type, for example changing a combo box to a set of radio buttons.

Form definition

The form tag contains a block of html fields that can be submitted to the server. <form> html is a simple way to generate the correct reference to ensure that a save action will run on the current page. Programmers may craft their own form tags if required. If programmers want to use GET method to send the form, the "method" parameter should be overridden. This tag must be closed with the </form> further down the html code.

Attributes
Example:
<P@Form @P> generates <form name="PagesName" method="post" action="/group/appname.pma/PageName?SavePage" >

Text field

Creates a new text form field.

Use: A simple one line text box for data input.

Attributes
name Defines the name of the text field
value The default value of the field. This value appears in the text field when in action form is not filled item with name as name attribute.
size Optional. The number of characters wide this field will appear on the page.
Example:
<P@Text name="txtField" @P> generates <input type="text" name="txtField" />

Password field

Creates a new password form field.

Use: Allows a secure input field for entering passwords

Attributes
name Defines the name of the form field
value The default value of the field. This value appears in the text field when in action form is not filled item with name as name attribute.
size Optional. The number of characters wide this field will appear on the page.
Example:
<P@Password name="pwfield" @P> generates <input type="password" name="pwfield" />

Text area field

Creates a new text area form field. This is field allows entry of paragraphs of text across multiple lines.

Attributes
name Defines the name of the form field
value The default value of the field. This value appears in the text field when in action form is not filled item with name as name attribute.
cols Number of columns in characters in the text area.
rows Number of rows in the text area
Example:
<P@TextArea name="text" value="" cols="40" rows="5" @P>

Checkbox field

Creates a checkbox field or group of checkboxes. Produces a set of boxes that may be checked on or off. None to all check boxes in the set may be checked.

Attributes
name Defines the name of the form field
value The default checked buttons of the group of checkboxes.
cols Maximum number of columns for checkboxes
choices The enumeration of checkboxes. This field has the same format for all listboxes, comboboxes, etc... The form is: option1|1,option2|2,option3|3.
usekeyword Gets the choices for this keyword from the named keyword
Example:
<P@Checkbox name="chk" choices="Choice Description|X" @P> generates <input name="chk" type="checkbox" value="X" />Choice Description

Radio buttons

Creates a group of radio buttons. Only one radio button in the set may be checked.

Attributes
name Defines the name of the form field
value Is the choice which is selected by default.
cols The maximum number of radio button columns.
choices Defines choices from which can user choose. The format is: option1|1,option2|2,option3|3.
usekeyword Gets the choices for this keyword from the named keyword
Example:
<P@Radio name="rad" choices="Choice1|1,Choice2|2" @P> generates <input type="radio" value="1" />Choice1 <input type="radio" value="2" />Choice2

List tag

Creates a combo box or list field. Combo field allows user to pick one of the provided possibilities. List allows to select more items at once. To display a list box, define attribute size, where size defines the number of rows of choices to display. To allow users to select multiple choices, specify multiple="1" attribute. Choices passed to the field can be not only listed in choices tag, but also in usekeyword attribute which loads choices automaticaly from keyword from Tornado system database.

Attributes
name Defines the name of the form field
value Is the choice which is selected by default.
choices Defines choices from which can user choose. The format is: option1|1,option2|2,option3|3.
multiple Optional: Allow selection of multiple values from a listbox.
size Number of rows in the listbox. Specifies that this tag is rendered as a listbox.
Example:
<P@List name="myList" choices="One|1,Two|2" @P> generates <select name="myList"><option value="1" >One</option><option value="2" >Two</option></select>

Date field

Creates a field which allows user to edit date. This date will be then passed as a Date object to the processing save action.

Attributes
name Defines the name of the form field
value Possible default values are: now, today, yesterday, tomorrow, nextweek, lastweek, nextmonth, lastmonth, nextyear, last year.
format Format of the date, for example dd.MM.yyyy. For a list of possible formats see http://java.sun.com/j2se/1.4.2/docs/api/java/text/SimpleDateFormat.html
Example:
<P@Date name="dtNow" value="now" format="yyyy-MM-dd" @P> generates <input type="text" name="dtNow" value="2006-06-28" />

Hidden field

Creates a text field which is not visible on the htmll page. This field should be used for example for transfering some identifiers in the form.

Attributes
name Defines the name of the field
value Defines the value to be placed in the field
Example:
<P@Hidden name="myID" value="2988" @P> generates <input type="hidden" name="myID" value="2988" />

Button

Creates a button.

Attributes
name Defines the name of the form field
value Defines text which appears on the button.
Example:
<P@Button name="btnSave" value="Save" onclick="doSave(this);" @P> generates <input type="button" name="btnSave" value="Save" onclick="doSave(this);" />

File field

Creates a file upload control which allows the user to select a file from their disk, and send it to the server. When this field is included on a page, the form tag is automatically changed to multipart.

Attributes
name Defines the name of the form field
Example:
<P@File name="importFile" @P> generates <input type="button" name="btnSave" value="Save" onclick="doSave(this);" />

Page parameter

When you pass a parameter to a page, this tag allows the programmer to insert that parameter value in the page without writing any page action. eg: if the page is opened with the url "http://server/group/app.pma&fred=17" everywhere in the page this tag appears with a name of "fred", will show the string "17"

Attributes
name Name of the parameter passed to the page.
Example:
<P@Parameter name="ParamName" @P>

Path

Shows the path to the current application. Useful for ensuring references to other application resources are explicit.

Attributes
value Optional. Values include: Group, Application, DesignElementName, Action, Parameters
Example:
<P@Path @P> generates /group/yourapp.pma/

Computed

The Computed tag is a placeholder which allow text to be inserted at that position from an action.

Attributes
name The nam ethat actions will use to insert text in this position.
value The text to be inserted.
usekeyword The keyword name to insert the keyword values
Example:
<P@Computed name="myField" value="8877" @P> generates 8877 ${TITLE}
